# Cognition-First 重设计提案

### 回应「流程太长太复杂 / 看不到 AI 亮点 / 没有体现 AI Cognitive」

> 内部设计讨论稿 · 2026-06-13
> 配套交付：高保真原型 `mmm-redesign-prototype.html`（独立文件，未改动现有 `frontend/` 代码）

---

## 一句话回应

**领导的批评成立，但这是一个信息架构（IA）问题，不是产品实质问题。** 产品里 AI Cognitive 的料其实很足——代码里已经有 22 条跨源推理发现（conflict / connection / reference）、8 个 C 类认知任务——只是被埋在角落的悬浮按钮里，而把最不该当主角的「确定性机械流程」摆在了正中央。**我们不缺 AI 的脑子，缺的是把脑子摆到台前。** 这一版重设计不需要重写，只需要把 IA 倒过来。

---

## 一、诊断：信息架构倒置了

把当前产品的「界面权重」和「真实价值权重」放在一起看，矛盾一目了然：

| 层 | 它的价值定位 | 当前界面位置 | 应该在的位置 |
|---|---|---|---|
| **机械流程（M/A，约 65% 任务量）** | 交付下限、必要但无差异 | **正中央**：左侧 33 节点 / 6 阶段任务清单，第一眼全是它 | 后台，折叠，只在需要时展开 |
| **认知洞察（C + Insight Engine）** | 差异化价值上限、AI 的「脑」 | **角落**：右下角悬浮按钮 → Tab → 才看得到 22 条 findings | 舞台中央 |
| **人的决策（H）** | 责任落点、信任来源 | 混在任务清单里，和机械步骤长得一样 | 显著、有专属视觉 |

这正好是我们自己那份《Value Reframing》文档反对的形态：文档说**价值在 C/H 节点和 Insight Engine（拉高上限），M+A 只是地板（保证下限）**——而当前 IA 把地板当成了门面。

三条批评，逐条对应到这个根因：

1. **「左边流程太长太复杂」** → 把 ~65% 的机械/自动步骤平铺成清单当头牌。这些步骤恰恰是「人不该盯」的部分，却占满了第一屏。
2. **「看不到 AI 亮点」** → 真正会让人「哇」的 AI 推理（跨源关联、口径矛盾、跨项目联想）被降级成悬浮 FAB，第一印象像项目管理工具，不像 AI 产品。
3. **「没有体现 AI Cognitive」** → C 类认知任务和 M 类机械任务在界面上长得**一模一样**（同一种任务行），AI 在「思考」这件事完全不可见。料在，但没被「演」出来。

**结论：实质没问题，是「主角和背景站反了」。修法是把 IA 倒过来，而不是推倒重来。**

## 二、重设计方向：Cognition-First Workstation（认知优先工作台）

一句话主张：**首屏不再回答「流程走到第几步」，而是回答「AI 想明白了什么、哪几件事需要你的判断」。** 机械流程退居为一条细进度轨，认知洞察占据中央。

新首屏 = 三个空间：

### 1. 左侧：把流程压成一条「进度轨」（解决批评 1）

- 6 阶段 → 一条竖向 **stage rail**（6 个点 + 当前阶段高亮），不再是 33 行任务清单。
- 机械/自动步骤**整段卷起**：一行 `AI 已自动完成本阶段 14 步 ✓`（可展开看明细），平时不占视觉。
- 只有 **C（需复核）/ H（需你决策）** 的任务才冒泡到轨道上，配数字徽标「3 件需要你」。
- 渐进披露：默认极简，想深入随时下钻——和业务上 L4→L5–L8 的下钻心智一致。

> 效果：左侧从「33 行机械清单」变成「6 个点 + 一句已自动完成 + 几件需要你」。长度和复杂度的观感直接砍掉一个量级。

### 2. 中央：Cognitive Cockpit（认知驾驶舱）——新的主舞台（解决批评 2、3）

把现在藏在 FAB 里的 Insight/Assistant **提升为首屏中央的英雄区**，呈现 AI 的「这个项目我读出了什么」：

- 大尺寸 **reasoning cards**（推理卡），按价值排序：矛盾 > 跨源关联 > 跨项目联想 > ROI 与先验冲突。
- 每张卡**把推理链显式画出来**（见第三节），不是给个结论让人盲签。
- 卡片直接带决策 CTA（采纳 / 转任务 / 起草客户问题 / 挑战），点完即落回流程。
- 这一区就是产品的「AI 亮点」——第一屏看到的不是流程，是 **AI 在思考**。

### 3. 右侧：证据 / 资产随上下文打开（保留，但服务于中央）

- 点推理卡上的证据芯片 → 右侧就地展开对应 Artifact（原文、血缘、来源），不跳走、不开抽屉。
- 资产从「独立目的地」变成「认知卡的放大镜」，让审阅闭环停留在一个屏里。

## 三、给 AI Cognitive 一套专属视觉语言（解决批评 3 的核心）

批评 3 的本质是：**C 类任务没有「长得像在思考」。** 解法是让认知卡和机械卡在视觉上彻底分家。

**机械卡（M）——刻意低调、可折叠：**
```
✓ 数据透视与粒度统一  ·  规则执行  ·  已自动完成
```
灰、扁、成组卷起，传达「这事不用你操心」。

**认知卡（C / Insight）——把「推理链」演出来：**
```
◆ 矛盾检测 · AI 较有把握
┌─────────────────────────────────────────────┐
│ TTS 投放口径跨年度含义不一致                          │
│                                                 │
│  观察   2024 与 2025 的 TTS spend 走势断层          │
│   ↓                                             │
│  假设   两年返点/佣金口径不同，直接入模会误导归因         │
│   ↓                                             │
│  证据   ① 数据字典 v3 · ② Sales 访谈纪要 8/14        │
│   ↓                                             │
│  建议   按年度拆分佣金口径后重算                       │
│                                                 │
│  [向客户起草口径确认]  [打开数据字典]  [挑战这个判断]      │
└─────────────────────────────────────────────┘
```
紫/蓝高亮 + 微光描边，竖向「观察 → 假设 → 证据 → 建议」的推理链，让人**看见 AI 怎么想到的**，而不只是它想到了什么。关键差异：

- **机械卡**回答「做完了什么」；**认知卡**回答「我推理出了什么、凭什么」。
- 认知卡必带 **「挑战这个判断」** 动作——这是体现 Cognitive 最强的交互：人能反问，AI 现场组装证据回应（先验区间、共线性 VIF、跨项目联想）。AI 敢被挑战，才像真的在思考。
- 不确定性写成措辞（「较有把握 / 存疑，建议人工复核」），而非埋头给个数字。

> 行为徽标随之分层：`已自动完成`（M，灰）·`AI 草稿`（A）·**`AI 分析`（C，高亮）**·`需要你决策`（H，琥珀）。让「AI 在动脑」的那一类一眼可辨。

## 四、五个可演示的「惊艳」时刻（全部用现有数据，零造假）

这些 finding 代码里**已经有了**，重设计只是把它们搬到中央、配上推理链。Demo 脚本直接可用：

| # | Wow moment | 类型 | 为什么惊艳 |
|---|---|---|---|
| 1 | **「8 月 O2O 销量异动 ↔ GM 访谈说 8 月平台补贴大战换了代理」** | connection | AI 把一条数据异常和一句访谈口供自动接上——人要翻两份材料才发现的，它一眼连起来 |
| 2 | **「TTS 投放口径跨年度含义不一致」** | conflict | AI 抓出人最容易忽略的「定义级」陷阱：数字对得上，口径对不上 |
| 3 | **「门店执行数据仅 5 个月，太短无法入模——上个饮料项目也栽在这」** | reference | AI 调用跨项目记忆，把别人踩过的坑提前预警 |
| 4 | **「冰柜 ROI 4.0 高于先验上沿，且与陈列架共线 VIF 6.2 → 建议合并变量重跑」** | challenge | 人质疑「这个数是不是太高了」，AI 现场组装证据反证并给修法 |
| 5 | **「关键问题 KBQ-A19（站外对站内影响）没有任何数据支撑」** | gap | AI 扫出 KBQ↔数据↔模型↔报告 链路的断点，主动补缺 |

**主推 #1 和 #2 作为 demo 开场**——它们最能在 5 秒内让人理解「这不是流程工具，是会读项目的 AI」。

## 六、落地分期（先验证、不动现有代码）

**Phase 0 · 原型对齐（现在，本提案配套）**
配套的 `mmm-redesign-prototype.html` 是独立高保真原型（用产品真实 tokens：slate + blue-600、Geist、radius 0.625rem），用来跟领导对齐方向，**不碰 `frontend/` 一行代码**。

**Phase 1 · IA 反转（最小改动、最大观感收益）**
- 首屏默认落到 Cognitive Cockpit（把现 `AssistantPanel` 的 findings 升为中央英雄区）；
- 左侧 `StageSpine` 之外，M/A 任务在 `ArtifactColumn`/任务列里默认折叠成「已自动完成 N 步」。

**Phase 2 · 认知视觉语言**
- 为 C 类 / Insight 新建 reasoning-card 组件（观察→假设→证据→建议 + 「挑战」动作）；
- 行为徽标分层，C 类高亮，M 类弱化。

**Phase 3 · 度量补齐**（呼应 Value Reframing 的缺口）
- 埋点 time-per-review、需人判断节点数、采纳率曲线，作为真正北极星。

> 注：Phase 1–3 是后续工程清单，**本轮只交付 Phase 0 的诊断、方案与原型**，按你的要求不改动现有代码。
