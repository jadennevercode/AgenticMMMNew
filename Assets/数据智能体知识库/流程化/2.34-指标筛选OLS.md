# 2.34 指标筛选 — OLS 快速验证（流程化 / 输入输出）

> 类别：Input/Output ｜ 用 OLS 快速验证「选哪个指标」能符合各 L4 因子的 Contribution 与 ROI Range。
> 是数据智能体技术线的**终选关**，承接 2.33（技术达标）与 2.31（业务区间），输出 2.35 Model Input。

## 为什么需要这一步

一个 L4 因子常有**多个候选指标**（如「品牌传播」有曝光量、花费；「社媒」有 KOL发帖数、声量、互动量、花费）。
即便都通过了 2.33 技术检验，仍需决定**入模用哪个**——判据是：哪个指标跑出来的 **Contribution / ROI 落在 2.31 业务预期区间内**。

## 流程

1. **【输入】** 2.33 通过的候选指标 + 2.31 各 L4 因子的 ROI Range / Contribution Range。
2. **【动作】** 对每个 L4 因子的候选指标分别做 **OLS 快速回归**（adstock + Hill + 线性回归的轻量版），估算其 Contribution 与隐含 ROI。
3. **【判定】** 选 Contribution / ROI **最贴合业务区间**的指标；偏离区间过大的候选剔除。
4. **【冲突处置】**
   - 若无候选落在区间内 → 回 2.31 复核业务假设 / 区间，或回 2.2 复核数据处理。
   - 若某因子所有候选都不可用 → **预警**该因子缺有效指标。
5. **【输出】** 每个 L4 因子的**选定指标**，写入 [`../模版化/2.35-Model-Input.md`](../模版化/2.35-Model-Input.md)，并分配 Variable/Metric 编号。

## 与最终建模（S3+）的关系

- 2.34 是"快速 / 单变量级"验证，目的是**选变量**，不是最终模型。
- 选定指标集合 = Model Input，交给 S3+ 做完整 MMM（多变量 adstock/Hill/OLS + Pareto 候选 + G4 模型选择）。
- 业务区间（2.31）在这里既是**筛选判据**，也会作为建模阶段的**先验 / 约束**延续使用。
