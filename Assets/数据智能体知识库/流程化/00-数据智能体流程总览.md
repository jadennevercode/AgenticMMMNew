# 数据智能体流程总览（2.1 → 2.35）

> 流程化内容：把数据智能体三个子阶段串成可执行的步骤蓝图。
> 每一步标注【输入】【动作】【输出】【交互/闸门】。

```
┌─────────────────────────────────────────────────────────────────────────┐
│ 2.1 数据校验 (Data Validation) ── 质量闸 1：原始数据能不能验收？           │
│   2.11 通用校验标准 + 打分规则(Input) ─▶ 2.12 数据质量评分(Output)          │
├─────────────────────────────────────────────────────────────────────────┤
│ 2.2 数据处理 (Data Process) ── ETL：打平到统一宽表                          │
│   2.21 宽表维度(In) + 2.22 处理逻辑(In·人确认) + 2.23 数据字典(I/O)         │
│        ─▶ 2.24 数据集(Output)                                              │
├─────────────────────────────────────────────────────────────────────────┤
│ 2.3 数据交叉校验 (Cross-Validation) ── 闸 2 业务 + 闸 3 技术 + 指标筛选     │
│   2.31 业务校验规则(In) ─▶ 2.32 数据展示/趋势解读(Out)                      │
│   2.33 技术校验规则(In) ─▶ 2.34 指标筛选 OLS(I/O) ─▶ 2.35 Model Input(Out) │
└─────────────────────────────────────────────────────────────────────────┘
```

## 阶段 2.1 — 数据校验

1. **【输入】** 因子树（来自 S1，L1–L4 + 指标）、各数据源原始文件、`2.11 通用校验标准` 与 `打分规则`。
2. **【动作】** 对每个 `(L1–L4 因子 × 指标)`，按 4 维（一致性 / 准确性 / 完整性 / 颗粒度）逐项打 0 / 0.5 / 1 分。
3. **【输出】** `2.12 数据质量评分表`：逐指标分维度得分 + Total Score + 文字说明。
4. **【闸门】** Total=1 验收；=0 该数据完全不能用、**预警**；=0.5 触发**人决策**（HITL）如何处理。

## 阶段 2.2 — 数据处理

1. **【输入】** 通过校验的数据源；`2.21 宽表数据集维度`（目标 Schema）；`2.22 数据处理逻辑`。
2. **【动作 a】** 建立主数据映射表（Product / Geo / Channel / Time 4 张主数据 + EC 渠道 / 省份城市 / 产品组 等 mapping 表），把每个源的维度清洗成统一名称，对不齐填 `NA`。
3. **【动作 b·人交互确认】** 对照 `2.22 TaskLog`（本案 37 个 task）逐 task 确认数据逻辑：task 编码/名称/优先级、源在 OSS 还是手工表、数据问题与反馈、processing logic。
4. **【动作 c】** 维护 `2.23 数据字典`：每个 source sheet 记录主题/来源系统/颗粒度/服务于哪个 L4 因子/ETL 逻辑/时间范围/X 或 Y/目标 ODS 表名/数据提供方。
5. **【输出】** `2.24 数据集`：所有源按统一维度集成的数据全集（`model_data` 长表）。

## 阶段 2.3 — 数据交叉校验

### 业务线（闸 2）
1. **【输入】** `2.31 指标业务校验规则`（六步业务分析框架 + 因子树 ROI/Contribution Range + 下钻维度）。
2. **【动作】** 六步：① 全景概览 → ② 维度拆解（渠道/产品/区域/营销活动）→ ③ 异常定位（下钻）→ ④ 假设生成 → ⑤ 先验提取（量化为模型参数约束）→ ⑥ 客户核对。
3. **【输出】** `2.32 数据展示`：Charting + 数据趋势解读 + **需与客户确认的数据问题清单**。

### 技术线（闸 3 + 筛选）
4. **【输入】** `2.33 指标技术校验规则`：CV（波动性）、Pearson（与 KPI 相关性）、VIF（因子间共线性）。
5. **【动作】** 逐指标算 Total = CV + Pearson + VIF；Good≥3 / Acceptable 1.5–3 / 不可用<1.5。**同一因子多指标时**，淘汰不达标指标；若某因子指标**全部**不达标 → **预警**。
6. **【动作·2.34 OLS 筛选】** 对剩余候选指标做 OLS 快速验证，选出贡献度（Contribution）与 ROI 落在 `2.31` 业务预期区间内的指标。
7. **【输出】** `2.35 Model Input`：由选定指标生成的最终建模输入数据（`model_data` 的选定子集 + Variable/Metric 编号）。

## 关键产物链（数据契约）

```
原始源文件 ─(2.1 打分)→ 验收清单 ─(2.21 宽表+2.23 字典)→ 统一长表 ─(2.24)→ 数据全集
   └─(2.31 业务/2.33 技术 交叉校验)→ 候选指标打分 ─(2.34 OLS)→ 选定指标 ─(2.35)→ Model Input
```
