# 2.35 Model Input（模版 / 输出 Schema）

> 类别：Output ｜ 数据智能体的最终交付物：由选定指标生成的建模输入数据。
> 出处：`Data Process_2.21&2.23 / Model Data`（model_data 的入模子集）。

## 它是什么

`2.24 数据集`（全集）经过 **2.31 业务交叉校验** + **2.33 技术统计检验** + **2.34 OLS 指标筛选** 之后，
**每个 L4 因子选定一个最优指标**，合成的最终入模长表。相对 2.24 多了建模编号列。

## 表结构（`vw_all_dataset_model_data_yyyymm`）

| 列 | 说明 |
| --- | --- |
| TASKNAME / 品牌 / 省份组别 / 省份 / 渠道类型 / 渠道 / 年 / 月 | 模型 + 时间颗粒度 |
| 数据源 | 来源 |
| 数据类型Level1–Level8 | 因子 L1–L4 + 下钻 L5–L8 |
| METRICS类型 / METRICS / VALUE | 指标与值 |
| **Variable no.** | 建模变量编号（如 `32`） |
| **Metric no.** | 建模指标编号（如 `32.1.1`） |

## 从 2.24 → 2.35 的收敛规则

1. 仅保留 **2.1 验收通过**（Total=1，或 0.5 经人决策保留）的数据。
2. 仅保留 **2.33 技术检验** Good / Acceptable 的指标。
3. 同一 L4 因子有多个候选指标时，**2.34 OLS** 选 Contribution / ROI 最贴合 `2.31` 业务区间的那个。
4. 因子全部候选指标不达标 → 不纳入，并在交付说明中**预警**该因子缺数据。
5. 为每个入模指标分配唯一 `Variable no. / Metric no.`，供建模阶段（S3+）引用。

## 与 backend 的衔接

- 该表即 `app/mmm` 的输入：adstock / Hill saturation / OLS 回归直接消费 `Model Input` 的 `(时间, 因子, 值)`。
- `Variable no. / Metric no.` 对应建模中的变量索引，确保贡献度分解可回溯到具体 L4 因子。
