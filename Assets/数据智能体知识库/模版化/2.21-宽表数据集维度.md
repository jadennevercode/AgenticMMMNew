# 2.21 宽表数据集维度（模版 / 核心数据契约）

> 类别：Input ｜ 数据智能体的「通用语言」：所有源 ETL 后打平到这套统一**长表（long format）**维度。
> 出处：`Data Process_2.21&2.23 / 宽表数据集维度`。维度 = 模型颗粒度 + 时间颗粒度 + 因子 L1–L4 + 下钻 L5–L8 + 指标类型定义。

## 设计要点

- **三段式维度**：①模型颗粒度（品牌·品类·产品·省份组别·省份·渠道类型·渠道）②时间颗粒度（年·月）③因子映射（L1–L4 业务因子 + L5–L8 下钻）。
- **主数据驱动**：四张主数据表（Product / Geo / Channel / Time）负责把异构源名称清洗成统一维度；**对不齐就填 `NA`**，绝不臆造。
- **长表而非宽表**：每行只承载一个 `(维度组合, 指标, 值)`，便于跨源 union、跨指标筛选与透视。
- **L1–L4 必须与因子树严格一致**，L5–L8 是该 L4 的下钻维度（如 by platform / by competitor / by region）。
