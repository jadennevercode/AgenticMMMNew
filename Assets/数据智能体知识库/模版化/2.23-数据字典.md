# 2.23 数据字典（模版 / 输入输出）

> 类别：Input/Output ｜ 记录每个 source sheet 的来源、口径、ETL 逻辑与去向。出处：`Data Process_2.21&2.23 / 数据字典`。
> 作用：数据血缘（lineage）+ 处理逻辑的单一事实源，是 2.2 数据处理的「施工图」。

## 字段定义（每行 = 一个 source sheet）

| 列 | 含义 | 示例 |
| --- | --- | --- |
| Theme | 数仓分层 | `ODS` |
| SheetName | 业务表名 | `EC Smartpath sales` |
| Source Sheet | 原始 sheet 名 | `ods_sales_..._gt_0112_年月` |
| Source Sheet 位置 | 物理位置 | `database` / `OSS://dan-cn-dbc-...` |
| 来源系统 | 系统来源 | `魔方` / `手工数据` |
| granularity | 数据颗粒度 | `monthly` / `yearly` |
| 服务于哪一个 L4 factor | 因子映射 | `竞品渠道成交驱动变化` |
| Source Sheet ETL逻辑 | 聚合/清洗逻辑 | `按照渠道和地域聚合` |
| 数据统计的时间范围 | 覆盖周期 | `202210-202510` |
| Y or X | 因变量/自变量 | `Y` / `X` / `X + Y` |
| Description | 说明 | 文本 |
| TableName | 目标 ODS 表名 | `ods_ec_data_yyyymm` |
| 表来源 | — | `线下表格` / `魔方` |
| Data source DEPT. | 数据归口部门 | `EC` / `Trade Mkt` / `SIA` / `Media` / `Cube` |
| Data provider | 提供方 | `Cube` / `EC` / `SIA` |

## 命名约定（从案例归纳）

- 目标 ODS 表统一前缀 `ods_`，后缀 `_yyyymm` / `_年` 表示分区粒度。
- 来源系统二分：`魔方`（数据库直取）vs `手工数据`（OSS 上的线下表格）。
- 每个源**必须**标注「服务于哪个 L4 因子」与「X / Y」，这是后续 2.31/2.33 交叉校验与建模选变量的依据。

## 文档使用指引（原表附带）

- `List`：所有表的索引页。
- `Change Log`：更改日志（日期 / 原因 / 版本 / 更改人 / 说明），手工维护。
- `数据表明细页`：每张表一个 sheet，含 Table/View Name、Theme、Display Name、Table Description 及列级 `Column Name / Datatype / 是否为KEY / FK To / Source_table / Logic / Comments`。

> 数据源全量清单（本案 ~90 个 source sheet → 数据集）见原始 Excel；本库不复制明细数据，仅固化 Schema 与约定。
