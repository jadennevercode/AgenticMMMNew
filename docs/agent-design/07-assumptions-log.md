> 目的：reference 材料不足以支撑产品设计的部分，由平台方按自身理解先行补充以推进设计，**但每条补充必须登记、可追溯、可推翻**。
> 与 [06-gaps-and-proposals.md](06-gaps-and-proposals.md) 的分工：06 记录**业务规则层**缺口（先验/调优等，需业务方回填规则）；本文档记录**产品设计层**假设（平台方已按假设先行设计，待确认后转正或返工）。

## 2. 假设登记（A 系列）

### A1 模型 Refresh / 持续运营未定义

- **材料现状**：reference 仅覆盖单次 POC 全流程，无模型刷新、持续报告的任何定义。
- **补充假设**：项目管控支持两类项目类型——`新建（Build）` 与 `刷新（Refresh）`。Refresh 决策规则：新增数据 ≤50% 且无新变量 → Refresh（沿用因子树与先验，重估系数）；否则 Rebuild（重走 S1–S4）。Refresh 项目跳过 S1 的因子召回，仅做纪要级增量更新。
- **依据**：MMM 行业通行的 refresh-vs-rebuild 启发式。
- **置信度**：中 ｜ **待确认人**：Solution Team
- **状态**：已按假设设计（05 文档项目类型预留）。

### A2 OLS 预验证（2.34）的判定标准无量化规则

- **材料现状**：StatusCheckList 仅一句"OLS 快速验证选择什么指标可以符合各 L4 因子的 Contribution 和 ROI Range"。
- **补充假设**：判定基准两层——①弹性/贡献落入行业知识典型范围（媒体 0.1–0.5、促销 0.5–1.2 等）；②与 2.31 形成并经客户核对的业务假设方向一致。偏离按红黄绿灯（<30% 黄、≥30% 红）处理，红灯指标回 2.31 重审或弃选。
- **依据**：行业知识.xlsx 弹性表 + 2.31 红黄绿灯规则的复用。
- **置信度**：中 ｜ **待确认人**：DS（Yifei 角色）
- **状态**：已按假设设计（02 文档 §4 2.34）。

### A3 Model Input（2.35）格式规范无文件

- **材料现状**：无格式定义。
- **补充假设**：转换规范 = 2.21 长表 Schema → 按模型设计声明（Model Design Spec：模型对象 × 层级 × 变量分组）透视为建模矩阵；每个模型对象一份输入（行=时间，列=KPI + 变量），附变量-因子树映射清单与缺失值处理记录。版本与长表版本、因子树版本绑定（血缘三元组）。
- **依据**：2.21 Schema + 2.32「6.MODEL Design」实例反推。
- **置信度**：高 ｜ **待确认人**：DS
- **状态**：已按假设设计。

### A4 访谈纪要总结框架（1.33）无模板

- **材料现状**：StatusCheckList 列为 Output 但无模板文件。
- **补充假设**：从 11 份 Danone 纪要反推标准三段式：`业务现状分享（含分享要点小标题）/ 数据情况 / 关键业务问题 & 假设`，头部元数据表（Date & Time / Theme / Attendee）。AI 按此框架生成纪要总结，并同时产出结构化抽取（因子树改动建议、数据口径备注、KBQ 候选）。
- **依据**：11 份纪要的实际结构高度一致。
- **置信度**：高 ｜ **待确认人**：BA（Sylvia 角色）
- **状态**：已按假设设计（01 文档 Step 4）。

### A5 交互问答（4.1 后半）边界未定义

- **材料现状**："标准模板 + 交互问答"仅六字。
- **补充假设**：问答范围限定在 `结果长表 + Tech Review + 因子树 + KBQ + 数据口径备注` 的检索域内；超出范围（如"帮我预测明年销量"）→ 明确拒答并转人工/转任务；ROI 类回答强制附"不可线性外推"提示；每个回答附证据锚点（引用图表/数据位置）。

- **置信度**：中 ｜ **待确认人**：BA + Solution Team

### A6 客户角色与权限模型未定义

- **材料现状**：客户在流程中是 sign-off 方与数据提供方，但材料未说明客户是否直接使用平台。
- **补充假设**：v1 客户**不直接登录**。顾问（BA/DS）代理操作：客户 sign-off 在线下 Review 会完成，顾问在平台录入决议并上传凭证（会议纪要/邮件截图）作为 Gate 证据。客户门户（自助 sign-off、数据上传、报告查阅）留作 v2。
- **依据**：2.32 趋势解读 deck 是"会议材料"形态；数据&指标沟通表由顾问代理往返。
- **置信度**：中 ｜ **待确认人**：产品负责人
- **状态**：已按假设设计（Gate 体系按"录入 + 凭证"建模）。

- **置信度**：低（纯平台方拟定） ｜ **待确认人**：项目负责人

## 3. 维护规则

1. 新假设一律在此登记后方可进入设计文档（设计文档中以"（假设 A_n）"标注引用）；
2. 确认/推翻时更新状态并在对应设计文档同步修订，推翻须登记返工范围；
3. 本表由项目管控智能体的 `change_tracker` 能力承载（上线后），当前人工维护。
