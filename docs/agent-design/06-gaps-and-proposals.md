## 3. 草案：3.1 先验设置规则

### 3.1 设计原则

1. **先验必须可追溯**：每条先验登记四元组 `来源假设（2.31 步骤4/5）→ 客户确认记录（步骤6）→ 数学形式 → 应用变量`；与"3A 业务总结"一一对应；
2. **分层取先验**：客户实验/历史模型 > 客户确认的业务判断 > 行业知识典型范围 > 弱信息先验；有真实实验（lift/geo test）时**必须**用于校准——未经校准的 MMM 与真值平均偏差约 25%（行业经验）；
3. **先验是约束不是答案**：宽度要保留数据说话的空间——客户给的是区间就用截断分布，给的是方向就只约束符号。

### 3.2 变量类型 → 先验家族

| 变量类型 | 适用对象 | 先验形式 | 默认参考 |
|---|---|---|---|
| 媒体类弹性 | L1 消费者需求驱动下各 L4 | 截断正态 Normal(μ, σ) 截断 [lo, hi] | 行业知识：媒体弹性 0.1–0.5（抖音 0.3–0.7、天猫 0.5–1.2 的渠道细分参考） |
| 促销弹性 | L1 促销优惠 | 截断正态，区分日常/大促 | 日常 0.5–1.2，大促 0.2–0.7 |
| 渠道执行/铺货 | L1 渠道成交驱动 | 正向截断 + 饱和点约束 | ND>80% 边际递减；SOS +1% → 销量 +0.5–1% |
| 竞品类 | L2 竞争格局 | 负向截断 Normal(μ<0, σ) 截断至负 | 实例：β_comp ~ Normal(−0.15, 0.05) |
| 品牌/搜索类（领先指标） | 品牌资产、社媒声量 | 滞后阶数先验（最大滞后 + 峰值位置） | 搜索领先销售 1–2 周，最大滞后 4 周 |
| 边际递减 | 所有投放类 | 弹性 = a − b×频次，b ~ Gamma(2,2)；或饱和曲线（Hill 类）形状参数区间 | — |
| 延迟效应（adstock/carryover） | 投放类 | 衰减率区间按媒介属性：长效品牌媒体（TV/OTV）衰减慢、效果类（投流/促销）衰减快 | 衰减率先验分媒介档位设置 |
| 季节/事件 | Seasonality Calendar、结构性突变（外卖大战） | dummy / 固定效应，"只 FOR 调模型校准"（唐老板规则） | 不参与 ROI 归因 |
| Base | 截距/基线 | **强约束为正**（Negative Base 是红旗） | — |

### 3.3 设置流程

1. 数据智能体提交先验提取清单（2.31 步骤 5 输出）；
2. 模型智能体按 §3.2 映射成数学形式，标注信息来源层级；
3. 缺客户确认的条目 → 生成核对问题（回 2.31 步骤 6 / 客户沟通表）；
4. G3.1 Gate：BA + 客户确认 → 先验登记表冻结（变更需重新走 Gate 并留痕）。

## 4. 草案：3.2 模型调优规则

### 4.1 调优目标（多目标，非单指标）

| 目标 | 度量 | 约束 |
|---|---|---|
| 拟合 | R²（benchmark 85–95%）/ MAPE（5–15%） | 不以牺牲业务合理性换拟合 |
| 残差健康 | DW 1.5–2.5；残差正态 p>0.05 | 异常按 3.3 处理路径修（加滞后/趋势/季节变量） |
| 业务合理 | 弹性落在先验区间；decomp 占比符合业务认知；无红旗 | Negative Base / 正向渠道负系数 = 一票否决 |
| 贝叶斯收敛（Meridian/PyMC 路径） | R-hat ≤ 1.01（全参数）；ESS 足够；无 divergence | 不收敛禁止出结果 |

### 4.2 调优操作目录（按诊断症状索引）

| 症状 | 允许的操作 | 禁止的操作 |
|---|---|---|
| 欠拟合（R² 低、残差有模式） | 补变量（回 2.34 候选池）、加滞后/季节项、调整函数形式 | 直接放宽先验去"凑"拟合 |
| 过拟合嫌疑（R² 极高 + 系数不合理） | 精简冗余变量（Adj R² 差距>0.1）、收紧先验、正则化 | — |
| DW <1.5 | 加 adstock/品牌资产/趋势变量 | 机械差分 |
| 共线性（2.33 VIF 提示） | 变量分组/合并（如 GDP、CCI 已被 NAB 反映则剔除——唐老板规则）、层级化处理 | 同时保留强共线变量各自出系数 |
| 弹性越界（偏离先验 >30% 红灯） | 与 BA 会商：①数据/口径问题 ②先验本身需修正（带证据重走 G3.1）③模型设定问题 | 单方面改先验让结果"过关" |
| MCMC 不收敛 | 增加采样量/调 sampler 参数、重参数化、简化层级 | 降低收敛标准 |

### 4.3 迭代纪律

- 每轮迭代登记：改了什么（变量/先验/超参）→ 为什么（诊断症状）→ 结果对比（五指标 + 红旗 + 弹性表）；由项目管控智能体归档（多轮 PROCESSING LOGIC 留痕的建模版）；
- 烟测（smoke run）结果不得用于业务结论，仅用于管线验证；
- **候选模型不自动定稿**：技术合格的候选集（含各自的检验报告与弹性对照表）提交 G3.4，由人按 ①业务合理性 ②校准/先验符合度 ③统计拟合 的优先级选择——与"最终模型选择是人的决策"原则一致；
- 模型版本与 Model Input 版本、先验版本三者绑定（血缘三元组），任何上游变更触发重训提示。

### 4.4 快速通道（OLS 模式）

需求是快速方向判断时允许 OLS 通道：跳过 3.1/3.2 的完整流程，但输出必须标注"快速估计，未经先验校准"，且不可直接进入 4.1 客户交付报告。
