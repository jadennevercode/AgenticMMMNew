# 第一部分 · Philosophy（页 2–4）
> 目标：先扭转高管对"AI Native"的常见误判，确立我们的价值框架。这一段是全篇的认知地基。

### 页 2 ｜ 放下"AI Native vs AI Embedded"的执念
- 常见误判：固定流程 → 没有自由规划空间 → Agentic 无法表达 → 只能 Embedded
- 错在哪：把"自由规划"当成了 Agentic 的本质
- 真正的本质：**基于当前状态，动态决定下一步该做什么**——这在固定流程里照样成立
- 一句话结论：**Agentic 是局部属性，不是全局属性**；MMM 这类驱动数千万预算、要求可解释的场景，Human-in-the-loop 不是缺陷，是信任与合规的前提

### 页 3 ｜ 换一个轴：从"架构标签"到"价值密度"
- 真正该问的不是"流程自不自由"，而是**能否把人的认知负荷持续往上抬**
- 同一套固定流程，差实现 vs 好实现：
  - 操作员 → 总审查官
  - 逐步操作 → 逐步审批 → **只看异常**
- 北极星指标：**单次审查耗时是否持续下降**（time-per-review）
- 核心隐喻：产品形态不是聊天框，而是 **"AI 提交 PR，人来 Review"** 的工作站

### 页 4 ｜ 我们的定位：能力门槛只降一档
- 不是"取代专家"，也不是"人人可用"，而是**把交付一份可信 MMM 所需的能力门槛，精确降低一档**
- Senior：吞吐倍增，一人顶一队，周期从周到天
- Mid-level：在 AI 辅助下，做到过去只有 Senior 能做的事——**TAM 安全扩张的真正来源**
- Junior/新手：仍达不到；他们缺的是判断力，而判断恰恰是当前最不该交给 AI 的部分
- 一句话：**"降一档"是价值，"一降到底"是风险**

# 第二部分 · 产品 Concept（页 5–7）
> 目标：展示我们如何把一条"确定性的固定流程"拆解分工、给每个节点配置自主性，从而真正变成 AI Native。

### 页 5 ｜ 确定性骨架，概率填肉
- 流程是固定 DAG：6 阶段 / ~33 节点（数据对齐 → 清洗 → adstock/饱和变换 → 模型设定 → 拟合诊断 → 业务校验 → 校准 → 预算优化 → 输出）
- 编排层刻意保持确定性，**自主性活在节点内部**——这正是"把 Agentic 从全局降为局部"的落地
- 一图：固定的流程画布（泳道 = Agent × 列 = 阶段），强调"序列固定、节点是判断的海洋"

### 页 6 ｜ 拆解分工：节点级四象限（M / A / C / H）
- 每个节点问两件事：是否需要判断 × 信息是否充分/风险高低，分四类：
  - **M 机械执行**：规则可完全执行，零 AI（让 LLM 算确定性的事既贵又不可靠）
  - **A 可自动**：AI 生成 + 机器可校验 + 人事后抽查
  - **C 认知判断**：需要思考但不担责 → AI 给建议+证据+候选，等人定夺（**最高价值区**）
  - **H 仅限人类**：责任不可委托（如最终选模，永不自动选）
- 公布配比 ≈ **45% M / 20% A / 22% C / 13% H**：自动化收益来自 M+A（约 2/3 工作量），差异化价值集中在 C 的质量与 H 的决策体验

### 页 7 ｜ 这就是 AI Native：四个放大动作
- **角色反转**：建模者从"跑一次模型"→"审查一份附带完整推理的草稿"
- **不确定性显性化**：关键输出带置信度与敏感性，而非孤零零的点估计（MMM 信任问题的命门）
- **主动异常路由**：AI 对照业务先验，判断"这 3 处需要人看"，把注意力精准导向 C 类节点
- **记忆带来一致性**：把建模者的隐性偏好沉淀为长期记忆，跨刷新周期复用——高频固定流程独有的红利
- 一句话收口：**AI Native 不是加了个 AI 按钮，而是让 AI 成为流程的运行方式**

# 第三部分 · 产品功能（页 8–10）
> 目标：把上面的理念翻译成高管能看见的产品能力，每页一个核心观点、图多字少。

### 页 8 ｜ 评审工作站（不是聊天框，不是黑箱仪表盘）
- 导航即工作站：项目三栏工作台 / 流程画布 / 决策 / 资产 / 知识；聊天降级为浮动助手
- HumanWorkbench：**一次只呈现一个待办**（"需要你"+"还有 N 个在排队"），无事时显示"AI 正在处理 X"的平静态
- 节奏由自主层级决定：M/A 一次跑完、批量事后抽查；只有 C/H 才阻塞等人——**绝不每步弹窗**

### 页 9 ｜ 决策卡 + 洞察引擎（核心交互资产）
- **DecisionCard**：结论先行（先给该判断的点）→ 证据 chips →每个选项都写明后果（"Then: …"）→ 决策语义的确认（"接受/按先验重拟/提问"，而非 OK/Cancel）
- **Proposal → Apply 协议**：AI 永不直接写状态，每次变更都是 结构化 diff + 理由 + 证据，过三道闸（schema → 规则 → 人）
- **Insight Engine**：主动产出 connection / gap / conflict / reference 四类发现，每条必须带可落回流程的"建议动作"（没有动作的洞察就是噪音）

### 页 10 ｜ 信任层：全局可追溯 + 可质询
- 全链路血缘：一键还原"AI 决定了什么、改了什么、为什么"，服务合规、也让人敢放手
- 工件生命周期：draft → proposed → confirmed → frozen；知识变更台账（增删合并 × 理由 × 来源 × 谁确认）
- "质询"动作：对任一数字追问（"这个弹性为什么这么高？"）→ AI 调取先验区间、共线性 VIF、跨项目召回作答
- 一句话：**AI 提议、专家审查、专家担责——责任链结构性闭合**
